## Быстрый старт
Для работы необходим установленный Docker.

```bash
git clone <repository-url>

# Создаем .env файл с настройками
make env

# Запускаем всю инфраструктуру
make up
```

## Доступ к сервисам, взаимодействие

- **swagger**: http://localhost:8080/swagger
- `make psql` для просмотра БД через утилиту psql

Так как в сервисе реализована аутентификация, для упрощения тестирования автоматически создан пользователь с кредами:
`admin@admin.com`
`admin`

Под этим пользователем можно тестировать API.

## Ключевые особенности
- Restful API
- Быстрая сборка и развертка приложения благодаря пакетному менеджеру `uv`
- Тесты через `pytest`, запуск командой `make test`
- Хранение секретов и работа через .env файл
- Разделение валидационной и сервисной логики
- Валидация входных данных
- Dependency Injection
- Изолированный CRUD
- doc-strings, type-hints, линтеры isort, flake8
- Сервис декомпозирован в виде контейнеров и разворачивается одной командой
- Написан в асинхронном стиле
- Используется alembic для миграций и при старте сервиса миграции автоматически применяются
- Используется механизм авторизации OAuth2
